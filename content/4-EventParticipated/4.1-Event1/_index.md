# AWS Workshop Reflection Report

### Purpose of the Workshop

- Share practical knowledge about operating and securing systems on AWS.
- Introduce a roadmap for preparing for the AWS Certified Cloud Practitioner certification.
- Improve understanding of Service Level Agreements (SLA), monitoring, and user experience.
- Explore AI-powered tools that automate application security.

### Speakers

- **Ngo Le Tan Huy** – AWS Cloud Practitioner Presenter
- **Nguyen Huynh Son** – Infrastructure Support Engineer
- **Nguyen Tuan Thinh** – DevOps/DevSecOps/Cloud Engineer

---

## Key Highlights

### AWS Cloud Practitioner

- Overview of the **AWS Certified Cloud Practitioner (CLF-C02)** certification.
- Introduction to the four exam domains:
  - Cloud Concepts
  - Security & Compliance
  - Cloud Technology & Services
  - Billing, Pricing & Support
- Explanation of the **AWS Shared Responsibility Model**.
- Review of essential AWS services, including:
  - Amazon EC2
  - Amazon S3
  - AWS IAM
  - Amazon VPC
  - AWS Lambda
  - Amazon RDS
- Useful preparation strategies:
  - Learning through keyword mapping
  - Practicing with the AWS Free Tier
  - Reviewing incorrect answers from practice exams

### SLA and Monitoring

- Understanding the concept of a **Service Level Agreement (SLA)**.
- The role of monitoring in risk management.
- Recognizing the difference between:
  - A healthy infrastructure.
  - A positive user experience.
- Introduction to the **Monitoring Pyramid**, including:
  - Cloud Provider
  - Infrastructure
  - Application
  - Business Metrics
  - Customer Experience
- Demonstration of AWS monitoring services:
  - Amazon CloudWatch
  - CloudWatch Alarms
  - Amazon SNS
- Emphasis on monitoring business metrics such as:
  - Login Success Rate
  - Checkout Success
  - Payment Success

### AWS Security Agent

- Challenges of traditional security testing:
  - Time-consuming
  - Expensive
  - Difficult to scale
- Introduction to the **AWS Security Agent**, powered by AI.
- Main capabilities:
  - Design Security Review
  - Code Security Review
  - Automated Penetration Testing
- Integration with GitHub and GitLab.
- Automatic vulnerability detection and code fix recommendations.
- Support for industry security standards:
  - AWS Well-Architected Framework
  - PCI DSS
  - NIST

## What I Learned

### AWS Knowledge

- Gained a clear understanding of the AWS Cloud Practitioner exam structure.
- Learned the use cases of core AWS services.
- Understood the Shared Responsibility Model between AWS and customers.
- Learned how AWS pricing models and cost management tools work.

### Monitoring and Reliability

- Monitoring should focus on more than CPU and memory usage.
- Business metrics and user experience are equally important.
- Learned how to build dashboards, alarms, and notifications using Amazon CloudWatch.

### Security

- Understood the importance of security throughout the software development lifecycle.
- Learned how AI can assist in application security reviews.
- Recognized the value of security reviews and penetration testing in DevSecOps.

---

## Applying the Knowledge

- Prepare for the AWS Certified Cloud Practitioner certification.
- Use Amazon CloudWatch and Amazon SNS to monitor system health.
- Focus on business metrics rather than infrastructure metrics alone.
- Incorporate security reviews during the system design phase.
- Integrate AI-powered security tools into the software development workflow.

## Workshop Experience

Participating in this AWS workshop provided valuable practical knowledge about cloud operations, system monitoring, and application security on AWS.

### Learning from Industry Experts

- The speakers shared valuable real-world experience in deploying and managing AWS solutions.
- Practical case studies made cloud concepts easier to understand and apply.

### Hands-on Technical Insights

- I gained a deeper understanding of the AWS Shared Responsibility Model.
- I observed live demonstrations of Amazon CloudWatch and Amazon SNS for monitoring and alerting.
- I learned that monitoring should focus on customer experience instead of infrastructure metrics alone.

### Exploring Modern Technologies

- I discovered the AWS Security Agent, an AI-powered solution for security reviews and automated penetration testing.
- I learned how AI can identify vulnerabilities and recommend code fixes during development.

### Key Takeaways

- A healthy infrastructure does not always guarantee a good user experience.
- Monitoring should prioritize business metrics and customer experience.
- Security should be integrated from the design stage of software development.
- Combining theoretical knowledge with hands-on practice and mock exams is the most effective way to prepare for AWS certifications.
